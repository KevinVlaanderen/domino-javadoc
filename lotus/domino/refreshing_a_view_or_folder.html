Refreshing a view or folder

Updates (additions, deletions, and changes) to a database are not reflected in an open view until it is refreshed. By default, a view refreshes automatically when a navigation method touches an updated document. You can suppress automatic refresh by setting AutoUpdate of NotesView to False. To explicitly refresh the view, call Refresh of NotesView

To refresh the UI view (the view rendition on the user screen), call ViewRefresh of NotesUIWorkspace. If the code is running in a UI document, call Refresh of NotesView first.
